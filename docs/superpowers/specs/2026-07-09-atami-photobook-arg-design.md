# 아타미 사진집 ARG — 설계 문서

작성일: 2026-07-09

## 개요

익명 게시판(2ch 스타일) 화면에서 진행되는 추리 ARG(대체현실게임). 플레이어는
게시판 스레드에 올라오는 글을 읽고, 중간에 나오는 퀴즈에 정답을 입력하면서
사건의 진상을 파헤친다. 진행 중 뉴스 기사·홍보 홈페이지 같은 "가짜 웹페이지"들이
링크로 열리고, 열어본 페이지는 오른쪽 **북마크 사이드바**에 종류별로 쌓인다.

사건 스토리는 이미 완성되어 있음(아타미 사진집 캠페인 → 모델 실종 → 부동산 사장
익사체 발견 → "사고사로 위장된 타살" 떡밥). **사진집 자체는 플레이어에게 오프라인으로
따로 제공**되므로 웹사이트는 사진집 뷰어를 포함하지 않는다. 웹사이트는 "캠페인 사이트"
쪽(게시판 + 뉴스 + HP + 북마크 + 정답 입력)만 담당한다.

이전에 만든 게임들처럼 **정적 HTML + GitHub Pages 배포** 방식.

## 목표 / 비목표

**목표**
- 스크린샷의 UI/느낌을 그대로 재현 (2ch 게시판, 뉴스지면, HP, 북마크 사이드바)
- 데이터 파일(`story.js`)만 채우면 게임이 완성되는 데이터 주도 엔진
- 텍스트 입력형 + 드롭다운 조합형 퀴즈를 하나의 유연한 틀로 처리
- 정답 시 다음 글이 시간 텀을 두고 한 문장씩 자동 등장
- 모바일 반응형, 한국어 UI

**비목표**
- 진행 상황 저장(localStorage) — **넣지 않음**. 매번 처음부터 플레이
- 사진집 뷰어 — **만들지 않음** (오프라인 제공)
- 서버/백엔드 — 없음 (완전 정적)

## 아키텍처

### 파일 구성
```
atami-arg/
  index.html      화면 껍데기 (메인 영역 + 북마크 사이드바)
  style.css       2ch 게시판 / 뉴스지면 / HP / 북마크 / 문제박스 디자인
  engine.js       범용 엔진 (렌더러 · 진행 관리자 · 퀴즈 판정기 · 북마크 관리자)
  story.js        사건 데이터 전부 (글, 문제·정답, 페이지, 잠금 규칙)
  images/         직접 제공하는 이미지 (뉴스 사진, HP 조감도 등)
```

### 엔진의 4가지 책임 (분리)
1. **렌더러** — 현재 열려있는 페이지(게시판/뉴스/HP)를 화면에 그림
2. **진행 관리자** — 몇 번째 글까지 열렸는지 / 어떤 문제를 풀었는지 추적, 정답 시
   다음 글·페이지 잠금 해제, 스레드 완료 시 다음 스레드 열기
3. **퀴즈 판정기** — 문제의 모든 칸(text 입력 / select 선택)이 정답인지 판정
4. **북마크 관리자** — 페이지가 처음 열릴 때 사이드바에 추가 + 토스트,
   목록 클릭 시 해당 페이지를 메인에 다시 표시

각 책임은 독립적으로 이해·테스트 가능하도록 분리한다.

## 데이터 구조 (`story.js`)

3층 구조: **스레드(장면) 순서 → 스레드 안의 글/문제 나열 → 링크로 열리는 페이지**.

### 화면(페이지) 종류
- **intro (오프닝 내레이션)**: 주인공 독백. "소문채널 열기" 버튼 → 게시판 톱
- **board (소문채널 게시판 톱)**: 항아리 로고 + 소개 문구 + 스레드 목록. 스레드
  제목 클릭 → 해당 스레드로 진입. (북마크에 "TOP"으로 표시)
- **thread (게시판 스레드)**: 글/문제 진행
- **news / hp**: 링크로 열리는 문서

### 게임 = 스레드(장면)들의 순서
한 스레드의 마지막 문제를 풀면 다음 스레드가 열린다. (현재 "사진집 캠페인 스레드"가
1탄. 나중에 데이터만 추가하면 2탄 가능)

### 스레드 = 글(post)과 문제(quiz)의 나열
- **글(post)**: 번호 · 익명 이름 · 랜덤 ID(예: `swb86w59c`) · 본문. 본문 안에
  다른 페이지로 가는 **링크**(빨간 배너 등)를 넣을 수 있음 → 누르면 그 페이지가
  열리고 북마크에 추가
- **문제(quiz)**: 이 지점에서 막힘. 정답을 맞혀야 그 아래 글들이 열림

### 페이지 = 링크로 열리는 뉴스/HP 문서
뉴스지면·홈페이지를 각각 하나의 문서로 정의(제목·날짜·문단·이미지). 링크로 열리고
북마크에 쌓인다.

### 문제(quiz) 모델 — "칸(field) 나열"
문제 = 질문 + 칸들의 나열. 각 칸은 셋 중 하나:
- **고정 텍스트** — 라벨·연결어 (`왜냐하면`, `와`, `의`, `가`)
- **입력칸(text)** — 직접 타이핑. 정답 여러 표기 허용(공백·표기 정규화 후 비교)
- **드롭다운(select)** — 목록에서 하나 선택. 정답 옵션 지정

**모든 칸이 정답이어야 통과.** 이 하나의 틀로 세 경우를 모두 표현:

| 예시 | 칸 구성 |
|---|---|
| `Q. 살인사건이 일어난 방 이름은?` | [입력칸] 하나 |
| `음식점 + 남성 + 만나고 있다` | [드롭다운][드롭다운][드롭다운] |
| `団体名 + 왜냐하면 [A]와 [B]의 [C]가` | [입력칸] + [드롭다운]와[드롭다운]의[드롭다운]가 |

## 화면 구성과 진행 흐름

### 레이아웃 (스크린샷과 동일)
- **가운데(메인)**: 현재 열려있는 페이지 표시 — 게시판 스레드 / 뉴스지면 / HP 중 하나
- **오른쪽: 북마크 사이드바** — 검정 손잡이 탭(🔖ブックマーク)으로 열고 닫힘.
  상단 종류 아이콘(🌐HP / 📝NEWS / 💬게시판), 아래로 열어본 페이지가 아이콘과 함께
  누적. 항목 클릭 → 그 페이지를 메인에 다시 표시
- **게시판 하단**: 현재 풀 문제 박스(회색 카드) + 입력칸/드롭다운 + 파란 `입력하는(入力する)` 버튼

### 진행 흐름
1. 스레드가 열리고 글들이 순서대로 보임
2. 문제 지점에 오면 아래 문제 박스 활성화 → 정답 입력
3. **정답** → 그 아래 새 글들이 **시간 텀을 두고 한 문장씩 자동 등장**(메신저 느낌).
   한 문장이 다 뜨면 잠시 후 다음 문장. 클릭 없이 진행
   - 글 속 링크(빨간 배너 등)를 누르면 뉴스/HP가 메인에 열리고 **북마크에 추가**
     ("북마크에 추가되었습니다" 토스트)
4. **오답** → "틀렸습니다" 안내 후 재시도
5. 스레드 마지막 문제까지 풀면 → **다음 스레드(장면) 잠금 해제**
6. 뉴스/HP 보다가 북마크 항목 또는 "게시판으로 돌아가기"로 게시판 복귀

### 순차 등장 세부
- 한 번에 열리는 글 묶음 안에서 **문장(줄) 단위**로 순차 표시, 문장 사이 짧은 딜레이
- 방금 새로 뜬 글은 살짝 하이라이트(연회색 배경)

## 디자인 / 스타일 (스크린샷 그대로)

- **게시판**: 흰 배경. 글 번호·`익명`·날짜·`ID:xxxx`는 초록색, 본문은 검정. 글 사이
  옅은 구분선. 새 글은 연회색 하이라이트
- **뉴스지면**: 큰 빨강 `News` 제목 + 진한 빨강 날짜 띠, 본문 문단, 사진 자리(캡션 포함)
- **HP(홈페이지)**: 홍보 페이지 느낌 (개발추진협회 등)
- **북마크 사이드바**: 오른쪽 검정 손잡이 탭 + 흰 목록 + 종류 아이콘 + 토스트
- **문제 박스**: 회색 카드 + 흰 입력칸 + 파란 `입력하는` 버튼
- **링크 배너**: 빨간 가로 배너
- 한국어 UI, 모바일 반응형

## 퀴즈 정답 판정 규칙

- **text 칸**: 입력값과 정답 후보들을 정규화(앞뒤 공백 제거, 내부 공백 제거, 전각/반각
  통일 등) 후 비교. 하나라도 일치하면 정답
- **select 칸**: 선택값 === 정답 옵션
- **통과 조건**: 문제의 모든 text/select 칸이 각각 정답
- **오답 피드백**: "틀렸습니다" 메시지, 입력값 유지하고 재시도 허용

## 미해결 / 나중에 정할 것

- 스토리 후반 "젖은 물건 2개 찾기"처럼 **여러 개를 골라내는 문제**: 현재 "칸 나열"
  모델의 select 여러 개로 표현하거나, 필요하면 다중선택(multi-select) 칸 타입을
  하나 추가. 해당 문제를 데이터로 옮길 때 결정
- "클릭하면 다음 문장 바로 표시"(빨리 넘기기) 옵션: 기본은 자동. 필요 시 추가
- 실제 이미지 파일명/치수: 사용자가 이미지 제공 시 `images/`에 넣고 story.js 경로 연결
```
