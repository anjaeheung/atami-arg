# 아타미 ARG 열림/닫힘 스위치 설계

날짜: 2026-07-10

## 목적
운영자(사용자)가 채팅으로 "오픈해"/"닫아"라고 지시하면, 배포된 게임을
플레이어에게 열거나 닫는 스위치. 닫으면 플레이어는 짧은 한 줄 안내만 본다.

## 방식 결정 (브레인스토밍 합의)
- 반영 속도: **1~2분 지연 허용** → 저장소 내 `status.json` + `git push`
  (별도 서버/비용 없음, 기존 배포 파이프라인 재사용)
- 닫힘 시 제작자 예외: **없음** — 닫으면 모두 닫힘 화면 (테스트는 재오픈 또는 localhost)
- 닫힘 화면: **짧은 한 줄**

## 구성

### 1. 스위치 파일 — `status.json` (저장소 루트)
```json
{ "open": true, "message": "현재 닫혀 있습니다." }
```
- `open` (bool): 열림/닫힘
- `message` (string): 닫힘 화면 한 줄. 생략 시 기본 문구 사용

### 2. 부팅 게이트 — `engine.js`
게임 초기화(부팅) 이전에 상태를 먼저 확인한다.
- `fetch("status.json?_=" + Date.now())` — 타임스탬프로 캐시 우회
- 응답 `open === false` → 닫힘 화면(한 줄) 렌더, 게임 부팅 안 함, 북마크 사이드바 숨김
- 응답 `open === true` → 평소대로 게임 부팅
- **fetch 실패/파싱 오류 → fail-open**: 게임 부팅 (일시적 오류로 전체 잠김 방지).
  드물게 닫힘이 안 먹는 사용자가 생길 수 있으나 팬게임 특성상 가용성 우선.

### 3. 닫힘 화면
- 게임과 동일한 어두운 배경, 중앙에 `message` 한 줄만.
- 전용 CSS 클래스(예: `.closed-screen`).

### 4. 전환 절차 (운영)
- "닫아" → `status.json`의 `open`을 `false`로 수정 후 `git push`
- "오픈해" → `open`을 `true`로 수정 후 `git push`
- GitHub Pages 빌드로 약 1~2분 뒤 반영

### 5. 부수 작업
- `engine.js` 수정에 따라 `index.html`의 버전 쿼리 `?v=2` → `?v=3`으로 상향(캐시 갱신).

## 비목표 (YAGNI)
- 즉시 반영(Cloudflare Worker) — 이번엔 안 함.
- 스케줄 자동 개폐, 접속자 수 제한, 예약 오픈 등 — 안 함.
